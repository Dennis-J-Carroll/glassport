# Glassport 0.6.9 — Security & Truthfulness (design)

**Date:** 2026-07-11
**Trigger:** GPT source review of the post-0.6.8 main branch (query-forward, 101
refusal, fail-closed redaction, private log perms). Six findings; #1 is a
**confirmed, executably-proven release blocker**. Analysis proposes a focused
security/truthfulness release rather than moving on to HTTP gating or replay.

**Governing invariant (unchanged, load-bearing):** *the relay is sacred.* No
change may alter, reorder, delay, or drop a wire byte on the happy path, and no
logging/analysis failure may kill a live session. Every PR proves it; where a
task conflicts with the invariant, the invariant wins.

**PyPI state:** 0.6.8 is live and carries the #1 secret-leak. This release
supersedes it.

---

## The confirmed blocker (finding #1) — evidence

The runtime PII scanner de-obfuscates before matching, but the scrubber replaces
by literal value against the *original* text. Detection succeeds; removal
silently no-ops.

```
_scan_pii(text):                       # detectors.py:918
    text = _normalize_for_scan(text)   # :920  strip invisible → fold confusables → NFKC
    ... value = m.group(...)           # :927  value is NORMALIZED (de-obfuscated)
    return [(pat, value), ...]

_apply_redactions(text, hits):         # :703
    if value and value in text:        # :705  value is normalized, text is ORIGINAL
        text = text.replace(value, ...) #        obfuscated → no literal match → SKIPPED
```

Executable repro (this session), an `sk-ant-…` key split with one U+200D:

```
scan hits: [('anthropic_key', 'sk-ant-api03-AAAAAAA')]     # detected
redacted output: 'sk-ant-api03-AAAA…‍AAAA…1234567890'  # secret intact
secret STILL RECONSTRUCTABLE: True
```

`redact_secrets_strict` only fails closed on *exception*; a no-op replacement is
not an exception, so the live key sails into the report/SARIF artifact. The HTML
report neutralizes deceptive chars for display *after* redaction, so the artifact
renders a fully reconstructable key with ‹U+200D› markers inserted — worst case
for a tool whose promise is truthful evidence.

## PR-A — span-aware redaction (unblocks release)

**Decision:** span-map + fail-closed backstop (chosen over span-map-only and
withhold-on-any-obfuscation).

### New: `_normalize_with_map(text) -> tuple[str, list[int]]`

Walk the **original char-by-char**, applying the same three stages as
`_normalize_for_scan` and recording, for each emitted normalized char, the
originating index in the source:

- invisible char (`_INVISIBLE_RE`) → emits nothing (deletion; no map entry)
- confusable (`_CONFUSABLES`) → 1 char out, maps to that origin index
- per-char `NFKC` → 0..N chars out, **all** tagged to that one origin index

`origin_idx[i]` = source index of normalized char `i`. `_normalize_for_scan`
becomes `_normalize_with_map(text)[0]` so the two can never diverge.

> **Known imprecision, made safe:** whole-string NFKC can combine *across*
> adjacent chars; per-char NFKC does not. The map is therefore a safe
> over-approximation of the original span (it may cover a slightly wider range —
> fine, it's a secret being deleted). The fail-closed backstop is the proof that
> imprecision never leaks.

### `_scan_pii` returns spans

It already computes `m.span(...)` at :930 for structural suppression. Extend the
returned tuple to carry the **normalized-coordinate** span alongside `(pat,
value)`. Existing consumers that want only `(pat, value)` get a thin wrapper.

### `_apply_redactions_spanned(original, hits_with_spans)`

For each normalized match `[a, b)` → original range
`[origin_idx[a], origin_idx[b-1] + 1)`. Redact the **original** span (covers the
embedded zero-width / homoglyph bytes) with `_redact(value, category)`. Apply
matches **right-to-left** so earlier offsets stay valid as the string shrinks.

### Fail-closed backstop (strict path only)

After span redaction, re-run `_scan_pii` on the output. If any hit survives,
return `_WITHHELD` — a visible placeholder is strictly safer than a live key in a
shareable artifact. `redact_secrets_strict` gets the backstop;
`redact_secrets` (lenient/internal) uses span redaction without it.

`neutralize_text` still runs *after* redaction on the non-secret remainder, so
deception **outside** secrets is still revealed. Truthful evidence preserved:
secret gone, server's use of a homoglyph elsewhere still surfaced.

### Grills (permanent, report + SARIF)

New adversarial cases, each asserting the **reconstructed** form is absent (not
just the plain string — today's grill checks only the plain string, which is why
a broken build passed):

- zero-width-split secret (U+200D between token chars)
- fullwidth / NFKC-compatibility secret
- Cyrillic / Greek homoglyph substitution
- multiple obfuscations inside one value

## PR-B — HTTP relay correctness (#2 + #3)

### #2 dynamic hop-by-hop filtering

HTTP lets a `Connection` header nominate extra hop-by-hop fields:

```
Connection: X-Internal-Hop
X-Internal-Hop: attacker-controlled     # must NOT forward; not in static _HOP
```

New `_hop_headers(headers) -> set[str]` = `_HOP` ∪ lowercased, validated tokens
parsed from **every** `Connection:` value (comma-split). Replace the static
`k.lower() not in _HOP` test in `_req_headers` (`mcp_http.py:110`) **and** the
response forward loop (`:320-324`) with this per-message drop set. Applied in
**both** directions (c2s and s2c). This is proxy correctness — hop-by-hop
headers are by RFC 7230 §6.1 **not** end-to-end forwardable, so removing them is
not a happy-path MCP-payload change.

### #3 strict remote-URL validation

`_validate_remote(url) -> SplitResult`, called in `run_http_tap` **before** the
listener binds:

- scheme ∈ {`http`, `https`} — reject everything else (no implicit "treat as HTTP")
- hostname required
- reject embedded `username` / `password` (userinfo)
- reject `fragment`
- validate port is in range
- build `Host` from **hostname (+ explicit non-default port)**, not raw
  `remote.netloc` (fixes `out["Host"] = remote.netloc` at `:111`, which forwards
  userinfo)

Invalid URL → clear CLI error, nonzero exit, listener never binds.

## PR-C — verified log perms + degradation (#4 + H5)

### `open_session_log(path) -> SessionLog | None` factory

Single creation path used by **both** stdio and HTTP taps. Wraps `SessionLog`
construction in try/except so **HTTP degrades exactly like stdio** — today the
stdio path catches `OSError` and sets `log = None` (tap.py:385) while the HTTP
path has no such guard and dies on an unwritable log dir. After open, verify with
`os.fstat(fd).st_mode & 0o777`: if the file is **not** owner-only (a silently
swallowed `fchmod` failure — `SessionLog.__init__:97`), emit a **loud** stderr
warning and return `None` (recording disabled, relay untouched).

`SessionLog` exposes the fd / verified mode so the factory can check it; the
in-`__init__` `try/except` around `fchmod` stays (never raise from the
constructor) but the swallowed failure is no longer invisible — the factory
catches it via `fstat`.

### `_NullLog` — dropped (YAGNI)

`pump(log: SessionLog | None, ...)` already tolerates `None` (tap.py:310). A null
object would be dead weight; return `None` and rely on the existing guard.

### Strict mode — deferred

A "refuse to operate when logging can't be secured" mode is optional per the
review. Default stays degrade-and-warn (relay is sacred). Add only on request.

### Relay-sacred proof

Client-byte **capture-diff** test: run a session with an unwritable log dir and
assert the client-visible byte stream is identical to the no-tap baseline
(logging off, forwarding intact).
